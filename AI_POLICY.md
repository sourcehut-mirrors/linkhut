# AI Policy

## This project

Generative AI tools have been used in the development of linkhut, primarily for research, learning, and code review,
with some code generation. The majority of the codebase was written before the advent of these tools and remains hand
coded. I want to be upfront about that, because I know it matters to some people. If you've taken a harder line on
AI-generated code and this is a deal breaker, I respect that.

## Contributing

Use of AI tools for contributing to linkhut is not banned. I've found them useful on occasion myself and won't ask
others to avoid them.

That said, I'd appreciate it if you disclose when a patch was AI-assisted. It helps me calibrate the level of review.
AI-generated code tends to need different scrutiny than code a person reasoned through line by line. This isn't a strict
requirement, since there's no reliable way to verify it, but I value the honesty.

## On the broader industry

This policy shouldn't be read as an endorsement of the generative AI industry, which raises serious concerns on many
fronts. On a personal level, I've spent considerable effort keeping AI scrapers from overwhelming ln.ht, which hasn't
exactly endeared me to the direction things are going.

## Subject to change

This policy may change over time, depending on how things play out: the review burden on patches, the state of the
tooling, or the broader landscape. I work on linkhut because I enjoy it, and a big part of that is understanding and
being familiar with every corner of the codebase. I won't accept changes from humans or machines that compromise that.
If the cost of AI-assisted contributions starts to undermine either my time or my enjoyment, the policy will tighten.
